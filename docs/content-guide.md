# コンテンツ設計ガイド (Content Guide)

**このファイルはPhase 2: Content作成時に参照してください。**

## カラースキーム（重要）

TOWINGスライドのカラーパレットは以下の比率で使用してください。

### 色の配分ルール（60-30-10ルール）

| 比率 | 色 | 用途 |
|-----|-----|------|
| **60%** | `#ab955f`（ゴールド）、`#f4f0e8`（クリーム） | ボックス背景、タイトル、見出し |
| **30%** | `#648651`（緑） | 強調色、太字テキスト、primaryボックス |
| **10%** | `#c46668`（赤） | **警告・アラームのみ**（warningボックス） |

### 自動適用される色

以下の色はシステムで自動設定されます：

| 要素 | 色 | 説明 |
|-----|-----|------|
| **スライドタイトル** | `#f4f0e8` | **クリーム（変更禁止）** |
| h2（セクションヘッダー） | `#ab955f` | ゴールド |
| h3（サブヘッダー） | `#ab955f` | ゴールド |
| **太字テキスト** | `#648651` | **緑（強調）** |
| 通常テキスト | `#333333` | ダークグレー |

**重要**: スライドタイトルの色（`#f4f0e8`）は絶対に変更しないでください。

### ボックス別の色

| ボックス | 背景色 | 文字色 | 用途 |
|---------|-------|-------|------|
| `::: box` | 白 + 緑ボーダー | 黒 | 一般情報 |
| `::: box highlight` | 白 + ゴールドボーダー | 黒 | 重要ポイント |
| `::: box primary` | `#648651`（緑） | `#f4f0e8`（クリーム） | **最重要メッセージ** |
| `::: box warning` | 白 + 赤ボーダー | 黒 | リスク・警告のみ |

### 緑ボックス（primary）内の強調

`.box.primary`内で強調する場合：
- 文字色: `#f4f0e8`（クリーム）固定
- 強調部分: **太字 + 下線**（色は変えない）

### 使用禁止のボックス

| ボックス | 理由 |
|---------|------|
| `::: box success` | 見た目が悪い（#22C55Eは使用禁止） |
| `::: box green` | 同上 |

### 赤色の使用制限

**赤色（`#c46668`）は以下の場合のみ使用してください**：

- `::: box warning` によるリスク・課題の強調
- 本当に注意を引く必要がある警告事項

**赤色を使ってはいけない場面**：
- 通常の強調（緑を使用）
- 重要メッセージ（primaryボックスを使用）
- 比較表のハイライト（highlightボックスを使用）

## 1スライド1メッセージの原則

各スライドには**1つの明確なメッセージ**が必要です。

### 良いスライドの構造

```
┌──────────────────────────────────────────────────┐
│  タイトル: トピック/カテゴリを示す                 │
│  （例: 「売上分析」「市場動向」「提案内容」）       │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌────────────────────────────────────────────┐ │
│  │  メッセージ（大きく強調）                    │ │
│  │  「売上は前年比30%増加し、目標を達成」       │ │
│  └────────────────────────────────────────────┘ │
│                                                  │
│  詳細・根拠・データ                              │
│  * 具体的な数値や事例                            │
│  * 背景情報や文脈                                │
│  * 補足説明                                      │
│                                                  │
└──────────────────────────────────────────────────┘
```

### メッセージの書き方

| NG | OK |
|----|----|
| 「売上推移」 | 「売上は3年連続で増加し、過去最高を更新」 |
| 「課題一覧」 | 「人材不足が最大の課題で、早急な対策が必要」 |
| 「提案」 | 「提案Bを推奨、コスト対効果が最も高い」 |

## コンテンツ密度レベル

### LOW（タイトルスライド、転換スライド）

- タイトル + サブタイトルのみ
- または タイトル + 1-2ボックス
- 本文の箇条書きは最小限

```markdown
# 第2部: 提案内容

::: box primary
**これからご提案する3つの施策について説明します**
:::
```

---

### MEDIUM（標準スライド）

- タイトル + 導入文（1-2文）
- ボックス（メッセージ）
- 箇条書き 5-6項目

```markdown
# 市場分析結果

国内市場は成熟期に入り、新たな成長戦略が求められています。

::: box highlight
**結論**: 海外展開とデジタル化が成長の鍵
:::

* 国内市場: 年平均成長率2%（横ばい）
* 海外市場: 年平均成長率15%（高成長）
* デジタル: 年平均成長率25%（急成長）
* 競合動向: 3社が海外展開を加速
* 顧客ニーズ: オンライン化への期待が高い
```

---

### HIGH（データ提示スライド）

- タイトル + メッセージボックス
- 2カラム または 詳細リスト
- 箇条書き 6-8項目
- 表やグラフの指示

```markdown
# 財務分析: 収益構造の変化

::: box primary
**重要**: 主力事業Aの利益率が低下、事業Bへのシフトが急務
:::

::: columns 50/50

::: column
::: box
**事業A（既存）**
* 売上: 100億円（前年比-5%）
* 利益率: 8%（前年10%）
* 市場シェア: 25%（横ばい）
* 課題: 価格競争の激化
:::
:::

::: column
::: box highlight
**事業B（成長）**
* 売上: 30億円（前年比+40%）
* 利益率: 15%（前年12%）
* 市場シェア: 10%（上昇中）
* 機会: 新規顧客開拓余地大
:::
:::

:::
```

## 最小コンテンツ量の基準

### 避けるべきパターン（スカスカ）

```markdown
# 課題

* 人材不足
* コスト増加
* 競争激化
```

→ **3項目だけでは情報が薄すぎる**

---

## 最大コンテンツ量の基準（重要）

### 避けるべきパターン（詰め込みすぎ）

スライドの領域をはみ出す量の情報は**絶対にNG**です。

**1スライドあたりの目安**:
- **箇条書き**: 最大6-7項目（ネストを含む場合は5項目まで）
- **ボックス**: 最大2-3個
- **テキスト総量**: 現在の70%程度を目指す

### 詰め込みすぎの兆候

以下の状態になったら情報を削減してください：

1. **スライド下部がはみ出す**: 表示領域を超えている
2. **ボックス内に箇条書きが5項目以上**: ボックスを分割する
3. **1つの箇条書きが2行以上**: 簡潔に言い換える
4. **文字が小さく感じる**: 内容を減らすか2スライドに分割

### 情報が多い場合の対処法

| 対処法 | 説明 |
|-------|------|
| **分割** | 1スライドを2スライドに分ける |
| **優先順位付け** | 最重要な3-4点に絞る |
| **階層化** | 詳細は補足スライドに移動 |
| **集約** | 類似項目をまとめる |

---

### 改善パターン

```markdown
# 当社が直面する3つの課題

::: box primary
**最優先課題**: 人材不足への対応が急務（離職率15%、業界平均の2倍）
:::

**課題の詳細**:
* **人材不足**: 技術者の採用難（応募数前年比-30%）、離職率上昇
* **コスト増加**: 原材料費20%上昇、人件費10%上昇
* **競争激化**: 新規参入3社、価格競争により利益率低下
* **デジタル対応**: システム老朽化、DX投資の遅れ
* **規制対応**: 2027年新規制への対応コスト（推定5億円）

::: box warning
**対策なしの場合**: 2027年に営業利益が半減するリスク
:::
```

## ボックスの正しい使い方

### ボックスの目的

ボックスは**重要なメッセージや結論を視覚的に強調する**ためのものです。
すべての情報をボックスに入れるのではなく、**本当に強調したい1-2文のみ**を入れてください。

### ボックス内の文字量制限

| ボックスタイプ | 最大文字数 | 用途 |
|--------------|----------|------|
| primary | 50-80字 | 最重要メッセージ、結論 |
| highlight | 50-100字 | 重要ポイント、示唆 |
| warning | 50-80字 | リスク、注意点 |
| 標準 | 100-150字 | 補足情報、まとめ |

### NG例：ボックスに詰め込みすぎ

```markdown
::: box
## 深刻化する構造的課題
* **就農者人口の激減と高齢化**:
  * 基幹的農業従事者数は10年で40%減少し122万人（2023年）
  * 平均年齢は68.4歳。70歳以上が過半数
* **耕作放棄地の拡大**:
  * 埼玉県に匹敵する約42万haが耕作放棄地に
  * 今後10年でさらに倍増するとの予測も
* **低い生産性**:
  * 農業従事者一人当たりの生産額は米独の約半分
  * 小規模・家族経営が中心で、大規模化が進まず
:::
```
→ **ボックス内に箇条書きを詰め込みすぎ。ボックスが意味をなしていない。**

### OK例：ボックスは結論のみ

```markdown
::: box warning
**構造的課題**: 就農者の高齢化と後継者不足により、今後10年で農業従事者は半減する見込み
:::

**課題の詳細**:
* **就農者人口**: 10年で40%減少し122万人、平均年齢68.4歳
* **耕作放棄地**: 約42万ha、今後10年で倍増予測
* **生産性**: 米独の約半分、小規模経営が中心
```
→ **ボックスは結論のみ。詳細はボックスの外に箇条書きで記載。**

### ボックスを分割する場合

内容が多い場合は、1つの大きなボックスではなく、複数の小さなボックスに分割：

```markdown
::: box highlight
**機会1**: スマート農業市場は年率15%で成長
:::

::: box highlight
**機会2**: 政府の大規模補助金により導入コストが低減
:::
```

## 情報が足りない場合の対処法

### 1. より具体的な情報を追加

| 抽象的 | 具体的 |
|--------|--------|
| 売上増加 | 売上30%増加（100億→130億円） |
| コスト削減 | 年間5億円のコスト削減 |
| 早期対応が必要 | 2026年Q2までに対応が必要 |

---

### 2. 背景・文脈を追加

- **なぜ重要か**: この課題が放置されると何が起こるか
- **歴史的経緯**: いつからこの状況か、どう変化してきたか
- **比較情報**: 業界平均、競合他社、過去との比較

---

### 3. 補足情報を追加

- **リスク/課題**: 実現に向けた障害
- **今後の展望**: 次のステップ、長期的な見通し
- **関連情報**: 他のプロジェクトとの関連

---

### 4. 画像/図のプレースホルダー

```markdown
::: box
**[図: 売上推移グラフ]**
* 種類: 折れ線グラフ
* X軸: 年度（2020-2026）
* Y軸: 売上（億円）
* 色: TOWINGブルー
:::
```

## 品質チェックリスト（詳細版）

### Outline段階

- [ ] スライド数は10-15枚か
- [ ] 各スライドの役割が明確か
- [ ] 論理的な流れ（問題提起→分析→提案→結論）があるか
- [ ] メッセージの一覧が作成されているか
- [ ] CTAスライドが含まれているか

### Content作成後

- [ ] 各スライドにメッセージがあるか
- [ ] メッセージはprimary/highlightボックスで強調されているか
- [ ] 箇条書きは最低5項目あるか（MEDIUMの場合）
- [ ] 具体的な数値が含まれているか
- [ ] 言語指定に従っているか（日本語/英語/併記）

### レイアウト確認

- [ ] 2カラムの場合、各カラム最低4項目あるか
- [ ] ボックススタイルの使い分けは適切か
- [ ] カラム比率は内容に合っているか

### 最終確認

- [ ] スライド下部に無駄な空白がないか
- [ ] 情報量のバランスは取れているか（薄いスライドがないか）
- [ ] タイトルとメッセージが矛盾していないか
