## ファイルヘッダ（メタデータ）

すべてのMarkdownファイルの冒頭に記述：

```markdown
***
title: "プレゼンテーション全体タイトル"
subtitle: "サブタイトル"
author: "著者名"
date: 2026-01-18
total_slides: 12
language: "ja"                    # ja / en / ja-en
bilingual: false                  # 日英併記の有無
target_audience: "経営層 / 技術チーム"
key_message: "3単語で伝えたい中心メッセージ"
tone: "corporate"                 # corporate / casual / technical
***
```

---

## スライド区切り（重要）

**各スライドの間には必ず `---` を入れてください。**

```markdown
***
slide_number: 1
layout: "title-slide"
***

# タイトル

---

***
slide_number: 2
layout: "outline"
***

# アジェンダ

---

***
slide_number: 3
...
```

**注意**: `---` がないと、全スライドが1つに結合されてしまいます。

### Slide 2: アウトラインスライド（必須）

**2ページ目は必ずアウトラインスライドを配置してください。**

```markdown
***
slide_number: 2
layout: "outline"
content_density: "medium"
***

# 本日のアジェンダ

<div class="outline-container">
<div class="outline-item"><div class="outline-number"><p>1</p></div><div class="outline-text"><p>背景と課題認識</p></div></div>
<div class="outline-item"><div class="outline-number"><p>2</p></div><div class="outline-text"><p>提案内容と解決策</p></div></div>
<div class="outline-item"><div class="outline-number"><p>3</p></div><div class="outline-text"><p>実行計画とタイムライン</p></div></div>
<div class="outline-item"><div class="outline-number"><p>4</p></div><div class="outline-text"><p>まとめと次のステップ</p></div></div>
</div>
```

**デザイン仕様（重要）**:
- 番号ボックス: `#ab955f`（ゴールド）背景の**正方形DIV**、28pt×28pt
- 番号テキスト: `#f4f0e8`（クリーム）、14pt太字、中央揃え
- 説明テキスト: 通常の黒文字
- 項目数: 3-5個が適切

**注意**: `<span>`ではなく`<div>`を使用すること。`<div>`のみがPowerPointの形状（Shape）に変換される。

**このスタイルは他のリストでも使用可能**:
- 手順説明
- 優先順位リスト
- フェーズ説明

## ボックススタイル一覧

| スタイル | 記法 | 用途 | 背景色 |
|---------|------|------|--------|
| 標準 | `::: box` | 一般的な情報 | 白（左ボーダー緑） |
| 強調 | `::: box highlight` | 重要ポイント、推奨案 | 白（左ボーダー金） |
| 警告 | `::: box warning` | リスク、課題、注意点 | 白（左ボーダー赤） |

## 表テンプレート（重要：積極的に使用すること）

**箇条書きで済ませず、構造化できる内容は表にしてください。**

### ネイティブ表オブジェクト（推奨: :::table記法）

PowerPointネイティブの表オブジェクトを生成します。**この記法を優先して使用してください。**

```markdown
:::table
{
  "headers": ["測定項目", "対照区（慣行）", "宙炭施用区", "改善率/変化"],
  "rows": [
    ["収量 (kg/10a)", "300", "335", {"text": "11.7%増", "style": "highlight"}],
    ["化学肥料施用量（N換算）", "100%", "50%", {"text": "50%減", "style": "highlight"}],
    ["葉の硝酸濃度（平均 mg/kg）", "2,500", "2,200", {"text": "12.0%減", "style": "highlight"}],
    ["生育日数短縮（日）", "55日", "50日", {"text": "5日短縮", "style": "highlight"}]
  ]
}
:::
```

### :::table 記法の詳細

**基本構造**:
```json
{
  "headers": ["列1", "列2", "列3"],
  "rows": [
    ["データ1", "データ2", "データ3"],
    ["データ4", "データ5", "データ6"]
  ]
}
```

**強調セルの指定**:
```json
["通常セル", "通常セル", {"text": "強調セル", "style": "highlight"}]
```

**カスタムスタイル（オプション）**:
```json
{
  "headers": ["列1", "列2"],
  "rows": [["データ1", "データ2"]],
  "style": {
    "headerBg": "AB955F",
    "headerFg": "F4F0E8",
    "rowAltBg": "F9F9F7",
    "highlightBg": "648651",
    "highlightFg": "F4F0E8"
  }
}
```

### 表を使うべき場面

| 内容 | 表にすべきか |
|-----|------------|
| 2つ以上の選択肢の比較 | ✅ 必須 |
| 数値データの一覧 | ✅ 必須 |
| 項目と説明のペア | ✅ 推奨 |
| 時系列の計画・ロードマップ | ✅ 推奨 |
| 単純な箇条書き | ❌ 不要 |

### 表のスタイル仕様（デフォルト）

- ヘッダー行: `#AB955F`（ゴールド）背景、`#F4F0E8`（クリーム）文字、太字
- 奇数行: `#FFFFFF`（白）背景
- 偶数行: `#F9F9F7`（薄いグレー）背景
- 強調セル: `#648651`（緑）背景、`#F4F0E8`（クリーム）文字、太字

### 廃止: HTMLテーブル記法

以下のHTMLテーブル記法は**廃止**されました。PowerPointで正しく表示されません。

```html
<!-- ❌ 使用禁止 -->
<table>
<tr><th>項目</th><th>内容</th></tr>
<tr><td>データ1</td><td>データ2</td></tr>
</table>
```

代わりに上記の `:::table` 記法を使用してください。

### パターン6: 課題分析フロー

原因と結果を構造的に示すレイアウト。

```markdown
# 大規模集約型農業の課題構造

高頻度な輪作体系は、土地利用効率と環境負荷の両面で課題を生じさせています。

::: box
**課題の根源**
* 高頻度かつ集約的な作付け（小松菜・ほうれん草）
* 土壌有機物の急速な消耗と病原菌の蓄積
* 硝酸態窒素（NO3-）の溶脱が容易な土壌特性
:::

::: box warning
**結果と経済的影響**
* 連作障害のリスク増大 → 土地稼働率の低下（最大のボトルネック）
* 肥料の利用効率（NUE）低下 → 定量化可能な経済的損失
* 地下水汚染・N2O発生 → 環境負荷増大（ESG評価への影響）
:::
```

**使用シーン**: 課題分析、因果関係の説明、問題提起
